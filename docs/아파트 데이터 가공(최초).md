## 1. 개요
본 문서는 수집된 아파트 원천 데이터(`apt_trade`, `apt_rent`, `apt_basic_info`, `apt_detail_info`)를 분석 및 Text-to-SQL 서비스에 적합한 형태인 최종 데이터셋(`apt_basic`, `apt_details`, `apt_rent_processed`, `apt_trade_processed`)으로 변환하는 상세 가공 절차를 기술합니다.

---

## 2. 상세 가공 절차

### 2.1 아파트 기본 정보 생성 (`apt_basic.csv`)
실거래가 데이터에 존재하는 아파트 단지들의 위치 정보를 확보하는 단계입니다.

1.  **대상 추출**: `apt_trade`, `apt_rent` 파일에서 유일한 `aptSeq`(단지 일련번호) 목록을 추출합니다.
2.  **주소 정제**: `sggNm` (시군구), `umbNm` (읍면동), `jibun` (지번), `roadNm` (도로명) 등을 조합하여 검색 가능한 주소 문자열(`search_addr`)을 생성합니다.
3.  **지오코딩 (Geocoding)**:
    -   Kakao Local API (`/v2/local/search/address.json`)를 호출하여 각 주소의 위도(`y`), 경도(`x`)를 확보합니다.
    -   좌표를 기반으로 Kakao Region API (`/v2/local/geo/coord2regioncode.json`)를 역호출하여 정확한 행정동 명칭(`admin_dong`)을 보정합니다.
4.  **저장**: `apt_basic.csv` 파일로 저장합니다.
    -   주요 컬럼: `apt_id` (aptSeq), `apt_name`, `road_address`, `latitude`, `longitude`, `admin_dong`

### 2.2 아파트 상세 정보 병합 (`apt_details.csv`)
K-Apt(공동주택관리정보시스템)의 상세 시설 정보를 실거래가 단지 ID와 연결하는 단계입니다.

1.  **데이터 병합**: 수집된 `apt_basic_info`와 `apt_detail_info`를 `kaptCode` 기준으로 Inner Join 합니다.
2.  **주차 대수 합산**: `kaptdPcnt` (지상) + `kaptdPcntu` (지하) = `total_parking` 컬럼을 생성합니다.
3.  **단지 매칭 (Linking)**:
    -   **1차 매칭 (이름+행정동)**: 아파트 이름과 행정동을 정규화(공백 제거, '아파트' 제외 등)하여 일치 여부를 확인합니다.
    -   **2차 매칭 (공간 매칭)**: 1차 매칭 실패 건에 대해, K-Apt 주소를 지오코딩한 후 `apt_basic`의 좌표와 유클리드 거리(반경 약 50m 이내)를 계산하여 가장 가까운 단지를 매칭합니다.
4.  **컬럼 표준화**: `apt_details_scheme_reference.csv`를 참조하여 분석에 필요한 컬럼만 남기고, 모든 컬럼명을 `snake_case` (Full Name)로 변경합니다.
    -   예: `kaptCode` -> `complex_id`, `kaptdaCnt` -> `household_count`, `doroJuso` -> `road_address`

### 2.3 전월세 데이터 가공 (`apt_rent_processed.csv`)
1.  **데이터 정제**:
    -   `excluUseAr` (전용면적)을 정수형으로 변환합니다.
    -   `dealYear`, `dealMonth`, `dealDay`를 결합하여 `deal_date` (YYYY-MM-DD) 컬럼을 생성합니다.
    -   `deposit`, `monthly_rent`의 컴마(,)를 제거하고 숫자형으로 변환합니다.
2.  **전세환산보증금 계산**:
    -   `rental_adjusted_deposit` = `deposit` + (`monthly_rent` * 12 / 0.045)
    -   전월세 전환율은 4.5%를 적용합니다.
3.  **시계열 분석 지표 생성**:
    -   동일 `apt_id` 및 `exclusive_area` 그룹 내에서 `deal_date` 순으로 정렬합니다.
    -   `previous_rental_adjusted_deposit`: 직전 거래의 환산보증금.
    -   `deposit_diff`: 현재 환산보증금 - 직전 환산보증금.
    -   `deposit_diff_rate`: 변동률 (%).
4.  **컬럼 표준화**: 불필요한 컬럼 삭제 및 Snake Case 적용.

### 2.4 매매 데이터 가공 (`apt_trade_processed.csv`)
1.  **데이터 정제**:
    -   `excluUseAr` 정수 변환, `deal_date` 생성.
    -   `deal_amount` (거래금액) 숫자형 변환.
    -   매수/매도자(`buyer_type`, `seller_type`), 거래유형(`dealing_type`) 등 상세 정보 유지.
2.  **시계열 분석 지표 생성**:
    -   동일 `apt_id` 및 `exclusive_area` 그룹 내에서 `deal_date` 순으로 정렬.
    -   `previous_deal_amount`: 직전 거래 금액.
    -   `deal_diff`: 현재 금액 - 직전 금액.
    -   `deal_diff_rate`: 변동률 (%).
3.  **컬럼 표준화**: 불필요한 컬럼 삭제 및 Snake Case 적용.

| :--- | :--- | :--- |
| **apt_basic.csv** | 위치 정보가 포함된 기본 데이터 | `apt_id`, `latitude`, `longitude`, `admin_dong` |
| **apt_details.csv** | 시설/관리 상세 정보 | `complex_id`, `apt_id`, `total_parking_count`, `household_count` |
| **apt_rent_processed.csv** | 가공된 전월세 데이터 | `rental_adjusted_deposit`, `deposit_diff_rate` |
| **apt_trade_processed.csv** | 가공된 매매 데이터 | `deal_amount`, `deal_diff_rate`, `buyer_type` |
